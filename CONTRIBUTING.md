# Contributing

Thank you for contributing to the **City4Climate DT Framework** repository.

This repository is intended to support the public dissemination and reuse of non-sensitive technical artefacts related to the project. Contributions should therefore remain aligned with the repository scope, dissemination rules, and documentation structure.

## Before contributing

Please ensure that any proposed contribution:

- is relevant to the repository scope;
- is suitable for public dissemination;
- does not contain personal data, confidential data, credentials, secrets, or restricted materials;
- is consistent with the current folder structure and documentation approach.

If the disclosure status of a file or dataset is uncertain, do not submit it until it has been validated.

## Types of contributions

Contributions may include:

- improvements to documentation;
- clarification of repository structure;
- source code approved for public release;
- example configurations and synthetic data;
- fixes to non-sensitive technical content;
- improvements to reuse and replication guidance.

## Recommended workflow

1. Create a branch from `main`.
2. Make focused and well-scoped changes.
3. Use clear commit messages.
4. Open a pull request with a concise description of:
   - what was changed;
   - why the change is relevant;
   - whether any dissemination or validation concerns exist.

## Documentation expectations

When contributing code or technical assets, please update the corresponding documentation where relevant, especially in:

- `docs/04_source-code/`
- `docs/05_reuse-guidelines/`
- `docs/06_demonstration/`

## Commit messages

Prefer short and descriptive commit messages, for example:

- `Add initial API documentation`
- `Update deployment instructions`
- `Add synthetic sample dataset`
- `Clarify reuse prerequisites`

## Pull request checklist

Before opening a pull request, confirm that:

- the contribution is suitable for a public repository;
- no sensitive or restricted content is included;
- the affected documentation has been updated where needed;
- file names and folder placement are coherent with the repository structure.

## Review and acceptance

All contributions are subject to maintainer review. Acceptance depends on technical relevance, documentation quality, and compatibility with the repository’s dissemination constraints.
