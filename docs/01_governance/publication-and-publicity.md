# Publication and Publicity

## Purpose

This document defines the initial public dissemination and publicity principles for the City4Climate DT Framework repository.

## General rule

All public-facing materials associated with this repository should comply with the communication, acknowledgement, and publicity obligations applicable to the project funding framework and to the project coordination rules in force.

## Publicity principles

Public outputs should:

- clearly identify the project name;
- acknowledge the funding framework in the wording approved by the project coordination;
- use official visual identity assets only when validated for public use;
- avoid inconsistent or unofficial funding references;
- remain aligned with any instructions received from the competent authorities and from the consortium coordination.

## Repository-level practice

For this repository, the following practice applies:

- the main `README.md` contains a concise funding and dissemination statement;
- public documents may include an acknowledgement note where appropriate;
- logos and visual assets should only be stored in `assets/logos/` after validation for public dissemination.

## Change control

This file is an initial baseline. It may be updated when formal publicity instructions, approved wording, or visual guidelines are confirmed by the project coordination and competent authorities.
